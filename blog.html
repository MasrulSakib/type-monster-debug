  1. 1. What is the difference between map, forEach, filter and find?


    map: Transforms each element of an array and returns a new array with the same length.
    forEach: Executes a provided function on each array element, but doesn't modify the array or return a new one.
    filter: Creates a new array containing elements that pass a provided condition.
    find: Returns the first array element that satisfies a condition, or undefined if none do.

  2. 2. What is the difference between local storage & session storage?


    Local Storage: It's a web storage mechanism that allows websites to store data as key-value pairs in a user's web browser. The data persists even after the browser is closed and can be accessed on subsequent visits to the same website. Local storage provides more storage space compared to cookies but is limited to the specific domain.
    Session Storage: Similar to local storage, session storage also stores data as key-value pairs in a user's browser. However, the data is only available for the duration of the browser session. Once the user closes the browser or tab, the session storage data is cleared.
    All three mechanisms are used for client-side data storage but serve different purposes and have varying capabilities and limitations.

  3. 3. How Eventloop works in Javascript/ES6?


    The event loop in JavaScript manages the order of execution for asynchronous tasks. It maintains a call stack for function execution, a callback queue for completed tasks, and utilizes microtasks (like promises) to prioritize certain tasks. This enables non-blocking behavior, efficient handling of I/O operations, and ensures responsive web applications.

  4. 4.What is the difference between global scope and block scope?


    Global scope refers to the accessibility of variables throughout the entire program, while block scope pertains to variables confined within a specific code block, like a function or loop. Global variables can be accessed and modified from anywhere, potentially causing unintended issues. In contrast, block-scoped variables are limited to the block where they're declared, enhancing code organization and reducing naming conflicts. This distinction is crucial for maintaining code clarity and preventing unintentional variable interactions.

  5. 5. How to get undefined in JavaScript?


    In JavaScript, you can encounter the value undefined in various situations: - When a variable is declared but not assigned a value. - When a function doesn't have a return statement or returns without a value. - When accessing a nonexistent property of an object. - When accessing an out-of-bounds index in an array. - By using the void operator followed by an expression, e.g., void 0. To intentionally get undefined, leave a variable uninitialized or use the void operator. It's essential to handle undefined appropriately to prevent unexpected behavior in your code.
